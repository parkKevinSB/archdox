Quick Start

연결은 네 단계면 충분합니다.

공개 문서는 흐름을 설명하고, 실제 secret과 사용량은 로그인 앱에서만 관리합니다.

1

앱 로그인

프로필 메뉴에서 개발자 / MCP 연결 화면으로 이동합니다.

2

연결 패키지 생성

Codex, Claude, Cursor, Custom Agent 중 연결 대상을 선택합니다.

3

설정 등록

생성된 MCP config를 AI 도구 설정에 붙여 넣습니다.

4

Live Smoke

4/4 성공이면 실제 gateway 연결과 scope가 정상입니다.

Client Config

연결 패키지는 이 형태의 MCP 설정을 만듭니다.

X-ArchDox-Engine-Key 값은 사용자의 secret입니다. 원문 키는 한 번만 표시되며, 잃어버린 키는 폐기 후 다시 발급해야 합니다.

추천 확인 방식 연결 직후 앱의 Developer 화면에서 live smoke를 실행하세요. 실패하면 오류 코드와 scope를 먼저 확인합니다. 
{
  "mcpServers": {
    "archdox": {
      "url": "https://mcp.archdox.co.kr/api/v1/mcp",
      "headers": {
        "X-ArchDox-Engine-Key": "adx_live_..."
      }
    }
  }
}

Document Review

PDF/문서 검토 흐름

외부 Agent가 문서를 던지고, ArchDox가 필요한 문맥을 더 묻고, 검토 결과를 구조화합니다.

review_inspection_document 문서 기반 검토 시작

감리일지 텍스트 또는 base64 PDF에서 대상 일자, 공종, 검사항목을 추출합니다.

추가 facts를 병합하고 같은 검토 세션을 다시 평가합니다.

ENGINE_REVIEW_SESSION
get_inspection_document_review_state 문서검토 상태 조회

안내 문장, 문맥 요약, 부족한 질문, 다음 MCP action을 조회합니다.

ENGINE_REVIEW_SESSION
validate_inspection_report 리포트 일괄 검토

문서 본문과 업무 facts를 제출하고 검증까지 한 번에 실행합니다.

ENGINE_REVIEW_SESSION

세션 primitive

Errors

실패 원인은 내부 코드와 함께 해석합니다.

ENGINE_API_KEY_INVALID 키가 없거나, 형식이 틀렸거나, 폐기/만료된 키입니다. 새 연결 패키지를 생성하세요.
ENGINE_API_SCOPE_REQUIRED 현재 키에 필요한 scope가 없습니다. 기본 키는 문서 검토, 법령 변경, 법령 검색 scope를 포함합니다.
ENGINE_API_QUOTA_EXCEEDED 일일 request unit 한도를 초과했습니다. Developer 화면의 사용량을 확인하세요.
MCP_INVALID_PARAMS tool arguments가 schema와 맞지 않습니다. Tool Catalog의 example arguments를 기준으로 수정하세요.
MCP_TOOL_NOT_FOUND 존재하지 않는 tool 이름입니다. tools/list 또는 Tool Catalog의 이름을 사용하세요.

Security

연결 키는 사용자의 책임 범위 안에서만 동작합니다.

Secret

원문 키는 한 번만 표시

ArchDox는 raw Engine API Key를 다시 보여주지 않습니다. 잃어버린 키는 폐기하고 새로 발급하세요.

Scope

Tool별 scope 분리

문서 검토, 법령 변경, 법령 검색은 capability와 scope를 통해 usage/quota가 분리됩니다.

Audit

호출 로그 기록

MCP 호출은 capability, toolName, request units, 실패 원인을 usage/audit 기록으로 남깁니다.

Ready

로그인 후 연결 패키지를 생성하세요.

Live smoke가 PASS이면 Codex, Claude, Cursor, Custom Agent에서 ArchDox MCP Gateway를 호출할 준비가 된 것입니다.
