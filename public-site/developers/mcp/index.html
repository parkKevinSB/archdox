MCP Gateway

MCP는 엔진이 아니라, ArchDox Engine을 호출하는 배포 채널입니다.

외부 AI Agent는 문서와 사실을 제출하고, ArchDox는 컨텍스트 정규화, 룰 선택, 검토 실행, 감사 기록을 책임집니다.

Architecture

AI가 도구를 직접 휘두르지 않게 합니다.

Customer AI Agent
  -> ArchDox MCP Gateway
  -> Context Intake
  -> Context Normalization
  -> Public Canonical Context
  -> Engine Policy Gate
  -> Flower-controlled Review Flow
  -> Structured Review Result

Planned tools

처음에는 검토 세션 기반 도구만 제공합니다.

create_review_session

검토 세션을 만들고 입력 범위, 기준일, 도구 권한을 고정합니다.

submit_document

문서 텍스트, 파일 메타데이터, 추출된 원문 조각을 제출합니다.

submit_context_facts

현장 조건, 공종, 구조, 용도 같은 추출 사실을 evidence와 함께 보냅니다.

normalize_context

고객 표현을 ArchDox 공용 canonical context로 정규화합니다.

run_validation

코드 검증, 룰 평가, 허용된 AI 검토를 실행합니다.

get_review_result

누락 항목, 근거, 수정 액션, 감사 메타데이터를 조회합니다.

Canonical context

외부 Agent에게 내부 DTO를 만들라고 요구하지 않습니다.

고객 Agent는 문서, 사실, evidence, confidence를 제출합니다. ArchDox가 정규화하고 내부 도메인 컨텍스트로 변환합니다. 

{
  "building": {
    "uses": ["NEIGHBORHOOD_LIVING_FACILITY"],
    "structureType": "REINFORCED_CONCRETE"
  },
  "construction": {
    "workType": "FOUNDATION_CONCRETE_PLACEMENT"
  },
  "document": {
    "type": "CONSTRUCTION_DAILY_SUPERVISION_LOG",
    "workDate": "2026-06-04"
  }
}
