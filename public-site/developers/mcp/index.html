MCP Gateway

AI 도구가 ArchDox Engine을 호출하는 공식 연결 경로입니다.

ArchDox MCP Gateway는 외부 Agent가 감리 리포트 검토, 법령 변경 조회, 법령 검색을 호출할 때 같은 인증, scope, quota, audit 경계를 타도록 하는 JSON-RPC/MCP endpoint입니다.

MCP endpoint https://mcp.archdox.co.kr/api/v1/mcp
로그인 후 연결 만들기 연결 순서 보기

Quick Start

가장 짧은 연결 순서

1

ArchDox 앱 로그인

사용자 앱에서 프로필 메뉴를 열고 개발자 / MCP 연결 화면으로 이동합니다.

2

연결 패키지 생성

Codex, Claude, Cursor, Custom Agent 중 연결 대상을 고르면 새 Engine API Key와 MCP config가 생성됩니다.

3

키와 config 보관

Engine API Key 원문은 한 번만 표시됩니다. 외부 AI 도구 설정에 넣고 안전한 곳에 보관하세요.

4

Live Smoke 실행

Developer 화면의 MCP 연결 테스트가 4/4 성공하면 실제 MCP endpoint 연결이 정상입니다.

Client Config

연결 패키지는 아래 형태의 MCP 설정을 생성합니다.

생성된 설정에서 X-ArchDox-Engine-Key 값은 사용자의 secret입니다. 공개 문서나 Git 저장소에 올리지 마세요. 기존 키 원문은 다시 조회할 수 없고, 잃어버리면 새로 발급해야 합니다.

추천 확인 방식 연결 후 app 개발자 화면에서 live smoke를 먼저 실행하세요. initialize, tools/list, get_legal_updates, search_law가 모두 성공해야 합니다. 
{
  "mcpServers": {
    "archdox": {
      "url": "https://mcp.archdox.co.kr/api/v1/mcp",
      "headers": {
        "X-ArchDox-Engine-Key": "adx_live_..."
      }
    }
  }
}

Tools

MCP V1 tool set

Tool catalog는 앱의 Developer 화면에서도 확인할 수 있습니다. 실제 quota/usage는 gateway에서 기록됩니다.

validate_inspection_report

리포트 일괄 검토

문서 본문과 업무 facts를 제출하고 검증까지 한 번에 실행하는 convenience tool입니다.

scope: ENGINE_REVIEW_SESSION
review_inspection_document

문서 텍스트 기반 검토

감리일지 텍스트 또는 base64 PDF에서 대상 일자, 공종, 검사항목을 추출하고 부족한 문맥 질문과 검토 결과를 반환합니다. 스캔 이미지 OCR은 다음 단계입니다.

scope: ENGINE_REVIEW_SESSION
answer_inspection_document_questions

부족한 문맥 답변

문서 검토에서 나온 review session에 건축물 용도, 구조, 현장 조건 같은 추가 facts를 병합하고 같은 검토를 다시 실행합니다.

scope: ENGINE_REVIEW_SESSION
create_review_session

검토 세션 생성

외부 Agent가 문서 검토 흐름을 시작할 review session을 만듭니다.

scope: ENGINE_REVIEW_SESSION
submit_document

문서 본문 제출

검토할 문서 텍스트와 메타데이터를 review session에 추가합니다.

scope: ENGINE_REVIEW_SESSION
submit_context_facts

업무 facts 제출

공종, 검사항목, 현장 조건, evidence 같은 구조화된 사실을 제출합니다.

scope: ENGINE_REVIEW_SESSION
normalize_context

문맥 정규화

외부 표현을 ArchDox canonical context와 카탈로그 기준으로 정리합니다.

scope: ENGINE_REVIEW_SESSION
run_validation

검증 실행

정규화된 context 기준으로 deterministic/source-backed 검증을 실행합니다.

scope: ENGINE_REVIEW_SESSION
get_review_result

검토 결과 조회

finding, legalReferences, nextActions, status를 같은 응답 계약으로 조회합니다.

scope: ENGINE_REVIEW_SESSION
get_legal_updates

법령 변경 조회

사용자에게 게시된 법령 변경 digest 목록을 조회합니다.

scope: LEGAL_UPDATES
explain_legal_change

법령 변경 상세

특정 digest와 원천 change set의 상세 내용을 읽습니다.

scope: LEGAL_UPDATES
search_law

법령 검색

동기화된 법령 코퍼스에서 후보 조문을 검색합니다. 후보만으로 최종 법률 판정을 내리지 않습니다.

scope: LEGAL_SEARCH
get_law_article

법령 조문 조회

source-backed article/version 기준으로 조문 본문과 출처를 조회합니다.

scope: LEGAL_SEARCH

Errors

실패 원인을 이렇게 해석하세요.

ENGINE_API_KEY_INVALID 키가 없거나, 형식이 틀렸거나, 폐기/만료된 키입니다. 새 연결 패키지를 생성하세요.
ENGINE_API_SCOPE_REQUIRED 현재 키에 필요한 scope가 없습니다. 기본 키는 문서 검토, 법령 변경, 법령 검색 scope를 포함합니다.
ENGINE_API_QUOTA_EXCEEDED 일일 request unit 한도를 초과했습니다. Developer 화면의 사용량을 확인하고 필요하면 한도 확장을 요청하세요.
MCP_INVALID_PARAMS tool arguments가 schema와 맞지 않습니다. Tool Catalog의 example arguments를 기준으로 수정하세요.
MCP_TOOL_NOT_FOUND 존재하지 않는 tool 이름입니다. tools/list 또는 Tool Catalog의 이름을 사용하세요.

Security

연결 키는 사용자의 책임 범위 안에서만 동작합니다.

Secret

원문 키는 한 번만 표시

ArchDox는 raw Engine API Key를 다시 보여주지 않습니다. 잃어버린 키는 폐기하고 새로 발급하세요.

Scope

Tool별 scope 분리

문서 검토, 법령 변경, 법령 검색은 각각 capability와 scope를 통해 usage/quota가 분리됩니다.

Audit

호출 로그 기록

MCP 호출은 capability, operation, toolName, request units, 실패 원인을 usage/audit 기록으로 남깁니다.

Ready

이제 로그인 후 연결 패키지를 생성하세요.
