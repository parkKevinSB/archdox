Recommended

먼저 런처를 설치하세요.

런처는 Agent runtime을 내려받고, 버전 호환성을 확인하고, 필요한 경우 업데이트합니다. 사용자는 런처만 유지하면 되고 Agent runtime zip을 직접 관리하지 않습니다.

Install Flow

다운로드와 계정 연결은 분리합니다.

01

런처 다운로드

공개 다운로드 페이지에서 OS에 맞는 런처를 내려받습니다.

02

앱 로그인

ArchDox 앱에서 개인 또는 사무소 작업공간을 선택합니다.

03

연결 토큰 발급

사무소와 장치가 연결될 수 있도록 일회성 토큰을 발급합니다.

04

저장소 선택

로컬 폴더, NAS, S3 호환 저장소 중 Agent가 사용할 위치를 정합니다.

Storage Policy

Agent는 하나지만 저장소 정책은 배포 방식에 따라 다릅니다.

개인 PC에 설치하면 로컬 폴더나 NAS를 쓸 수 있고, ArchDox가 관리하는 클라우드 Agent는 S3 호환 저장소를 사용합니다. 운영 배포에서는 문서/사진 저장소와 런처 배포 버킷을 분리합니다.

Local Office Agent 사무소 PC, NAS, 로컬 폴더 중심 운영
Cloud Managed Agent ArchDox 서버에서 구동, S3 호환 저장소 사용
