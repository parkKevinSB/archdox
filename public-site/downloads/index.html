앱에서 계정 연결

설치 후 ArchDox 앱에서 개인 또는 사무소 작업공간을 선택하고 연결 토큰을 발급합니다. 아무 Agent나 계정에 붙지 못하도록 장치 단위로 승인합니다.

설치 흐름

다운로드와 계정 연결은 분리합니다.

1. 런처 다운로드 공개 페이지에서 Windows 런처를 내려받습니다. 2. 앱 로그인 개인 또는 사무소 작업공간을 선택하고 연결 토큰을 발급합니다. 3. 저장소 설정 로컬 폴더, NAS, S3 호환 저장소 중 Agent가 사용할 위치를 정합니다.

개인 PC 설치 가이드

Windows PC에서 먼저 실행 확인까지 진행합니다.

현재 배포는 설치 마법사가 아니라 명령 실행형 런처입니다. 아래 순서대로 진행하면 런처가 필요한 Agent 실행 파일을 내려받고, 로컬 PC에서 Agent 상태를 확인할 수 있습니다.

1

다운로드한 zip 압축을 풉니다.

예시는 C:\ArchDox 아래에 런처를 두는 기준입니다.

mkdir C:\ArchDox
Expand-Archive .\archdox-agent-launcher-windows-x64-0.0.1.zip C:\ArchDox\agent-launcher -Force
cd C:\ArchDox\agent-launcher\archdox-agent-launcher\bin
2

Agent 실행 파일을 설치합니다.

런처가 ArchDox 서버에서 Agent 실행 파일을 내려받고 무결성을 확인합니다.

.\archdox-agent-launcher.bat `
  --cloud-api-base-url https://api.archdox.co.kr `
  --apply-update `
  --install-dir C:\ArchDox\agent-runtime `
  --work-dir C:\ArchDox\agent-launcher-work
3

Agent를 시작합니다.

시작 후 로컬 상태 확인 주소가 `UP`이면 Agent가 실행 중입니다.

.\archdox-agent-launcher.bat `
  --cloud-api-base-url https://api.archdox.co.kr `
  --launcher-command start `
  --install-dir C:\ArchDox\agent-runtime `
  --work-dir C:\ArchDox\agent-launcher-work `
  --runtime-health-url http://127.0.0.1:18080/actuator/health
4

상태를 확인합니다.

브라우저에서 아래 주소를 열었을 때 `UP`이 보이면 로컬 Agent가 살아 있습니다.

http://127.0.0.1:18080/actuator/health
계정 연결은 앱에서 진행합니다. 설치 파일 실행과 실제 계정 연결은 분리되어 있습니다. 앱에서 개인 또는 사무소 작업공간을 선택하고, 연결 토큰과 저장소 위치를 설정해야 문서 생성 작업에 사용할 수 있습니다.
