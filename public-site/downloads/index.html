앱에서 계정 연결

설치 후 ArchDox 앱에서 개인 또는 사무소 작업공간을 선택하고 연결 토큰을 발급합니다. 아무 Agent나 계정에 붙지 못하도록 장치 단위로 승인합니다.

설치 흐름

다운로드와 계정 연결은 분리합니다.

1. 런처 다운로드 공개 페이지에서 Windows 런처를 내려받습니다. 2. 앱 로그인 개인 또는 사무소 작업공간을 선택하고 연결 토큰을 발급합니다. 3. 저장소 설정 로컬 폴더, NAS, S3 호환 저장소 중 Agent가 사용할 위치를 정합니다.

개인 PC 설치 가이드

한 번 등록하면 Windows 로그인 때 자동으로 실행됩니다.

Windows용 패키지는 Agent 런처와 작은 Java runtime을 함께 포함합니다. 압축을 풀고 설치 파일을 한 번 실행하면 Windows 로그인 때 자동으로 Agent가 다시 켜집니다.

1

다운로드한 zip 압축을 풉니다.

예시는 C:\ArchDox 아래에 푸는 기준입니다. 다른 폴더에 풀어도 됩니다.

mkdir C:\ArchDox
Expand-Archive .\archdox-agent-launcher-windows-x64-0.0.1.zip C:\ArchDox\agent-launcher -Force
cd C:\ArchDox\agent-launcher\archdox-agent-launcher
2

설치 파일을 실행합니다.

아래 파일을 더블클릭하면 runtime 설치와 Windows 자동 실행 등록이 같이 진행됩니다.

Install ArchDox Agent.bat
3

재부팅 후에도 자동 실행됩니다.

PC를 껐다 켜도 Windows 로그인 후 작업 스케줄러가 Agent를 다시 실행합니다. 사용자가 매번 켤 필요가 없습니다.

Get-ScheduledTask -TaskName "ArchDox Agent"
Start-ScheduledTask -TaskName "ArchDox Agent"
4

상태 확인과 제거도 가능합니다.

브라우저에서 `UP`이 보이면 실행 중입니다. 자동 실행을 끄려면 제거 파일을 더블클릭합니다.

http://127.0.0.1:18080/actuator/health
Remove ArchDox Agent.bat
Java를 따로 설치하지 않아도 됩니다. 패키지 안에 Agent 런처 실행에 필요한 작은 Java runtime이 들어 있습니다. 다음 단계에서는 이 과정을 EXE/MSI 설치 마법사와 로컬 설정 화면으로 더 자연스럽게 감쌀 예정입니다.
